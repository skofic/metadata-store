# `_key`

**`_title`**

Document key

**`_definition`**

The unique identifier of a document within its collection. In term documents it is set by the loader from the term's global identifier; in edge documents it is computed from the MD5 hash of the source, predicate, and destination.

**`_description`**

The document key is the primary key of a record within its collection in ArangoDB. Together with the collection name it forms the [document handle](_id.md) (`_id`).

In this dictionary the key is never supplied by the user or auto-generated by the database — it is always computed:

- For [term documents](_term.md), the loader derives `_key` from the term's [global identifier](_gid.md): the [`_gid`](_gid.md) value is copied to `_key` before the document is saved.
- For [edge documents](_edge.md), `_key` is the MD5 hash of the concatenation `_from + "/" + _predicate + "/" + _to`.

Once set, the key is immutable.
