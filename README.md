# Metadata Store

**A database-hosted metadata repository and data dictionary**

---

## Overview

This repository defines a self-describing [data dictionary](https://en.wikipedia.org/wiki/Data_dictionary) and metadata repository hosted in an [ArangoDB](https://www.arangodb.com/) NoSQL graph database. It provides a formal, structured vocabulary for describing datasets, variables, and data standards — including their types, units, valid ranges, controlled vocabularies, and relationships.

The dictionary is **self-describing and recursive**: the structures used to define terms are themselves terms in the dictionary. This makes the ontology navigable and machine-readable without external schema files.

The repository contains the source data, documentation, and tooling needed to populate the dictionary with its core building blocks and ancillary data standards.

---

## What This Repository Contains

### `data/core/` — Core dictionary terms

The foundational terms that define the dictionary's own structure. These are the building blocks from which all other terms are defined:

| File | Contents |
|---|---|
| `containers.json` | Top-level structural terms: the term object, the edge object, and the six term sections (`_code`, `_info`, `_data`, `_rule`, `_prop`, `_domn`) |
| `_code.json` | Identifier properties: `_gid`, `_lid`, `_nid`, `_aid`, `_pid`, `_name`, `_symbol`, `_regexp`, `_emoji` |
| `_data.json` | Data shape properties: `_scalar`, `_array`, `_set`, `_tuple`, `_dict`, and their sub-structures |
| `_scalar.json` | Scalar descriptor properties: `_type`, `_kind_*`, `_unit`, `_regexp`, `_decimals`, range properties |
| `_scalar.terms.json` | Enumeration elements for scalar types and qualifiers: `_type_*`, `_kind_number_*`, `_kind_string_*`, `_kind_key_*` |
| `_rule.json` | Schema rule properties: `_required`, `_recommended`, `_banned`, `_closed`, `_computed`, `_locked`, `_immutable`, `_default-value`, and selection-descriptor terms |
| `_domn.json` | Domain/role classifier property: `_kind_term` |
| `_domn.terms.json` | Enumeration elements for term roles: descriptor, namespace, enumeration root/element, object definition, predicate |
| `_predicates.json` | Graph predicate terms: `_predicate`, `_path`, `_path_data`, and all built-in predicates |
| `_predicates.edges.json` | Edge documents forming the predicate enumeration graph |
| `_scalar.edges.json` | Edge documents encoding value-conditional rules for the `_scalar` structure |
| `database.json` | ArangoDB collection and graph definitions |

### `terms/` — Generated term cards

Markdown documentation cards automatically generated from the source JSON files, one per term. These provide human-readable reference documentation for every term in the dictionary. Generated by the workflow in `workflows/term-cards/`.

### `workflows/term-cards/` — Term card generator

A Swift command-line tool that reads all `*.json` files in `data/core/` and writes a Markdown documentation card for each term into `terms/`. Run with:

```bash
cd workflows/term-cards
swift run
```

## Term Structure

Every term is a JSON document with up to six top-level **sections**:

| Section | Purpose |
|---|---|
| `_code` | Identifiers: global ID (`_gid`), local ID (`_lid`), namespace (`_nid`), and aliases |
| `_info` | Multilingual human-readable content: title, definition, description, examples |
| `_data` | Data type and shape description (makes a term a *descriptor*) |
| `_rule` | Object schema rules: required, recommended, banned, computed, and default-value properties |
| `_prop` | Concrete attributes of the real-world entity the term represents |
| `_domn` | Categorical classification of the term's role in the dictionary |

The `_gid` (global identifier) is computed as `_nid + "_" + _lid` and doubles as the ArangoDB `_key`, enforcing global uniqueness. Terms whose `_nid` is an empty string are core dictionary building blocks (their `_gid` begins with `_`).

---

## Graph Model

Relationships between terms are stored as **edge documents** in ArangoDB edge collections. Each edge has:

- `_from` / `_to` — the related nodes
- `_predicate` — the type of relationship (enumeration membership, property of, value of, etc.)
- `_path` — the set of graph root handles that route through this edge (enables graph sharing without edge duplication)
- `_path_data` — context-sensitive data attached to the edge, keyed by graph root or node handle

This model supports enumeration graphs, object property graphs, alias resolution, bridge graphs (vocabulary subsets), and value-conditional validation rules — all within a single unified edge structure.

## Planned Development

This repository is the first in a three-project suite hosted under the [skofic](https://github.com/skofic) organisation:

### 1. Metadata Store *(this repository)*
- [x] Core term documentation and source JSON
- [x] Term card generator
- [ ] Validation library (JavaScript/TypeScript) — validates values and objects against dictionary rules, including conditional graph-based rules
- [ ] Database loader — populates an ArangoDB instance from the source JSON files

### 2. [Foxx Services](https://github.com/skofic/metadata-foxx) *(planned)*
ArangoDB [Foxx](https://www.arangodb.com/docs/stable/foxx.html) microservices providing a REST API over the dictionary database. Deployable directly from its GitHub URL into any ArangoDB installation. Will depend on the validation library from this repository.

### 3. [Data Dictionary UI](https://github.com/skofic/metadata-ui) *(planned)*
A web application for navigating the data dictionary as an ontology — following relationships between terms, exploring controlled vocabularies, and viewing term documentation. Will also provide:
- Tools for adding and editing terms with guided forms
- Dataset submission and navigation
- Breadcrumb-style navigation history

## Data Standards (Planned — Phase 3)

The dictionary will include terms for widely-used international standards, sourced primarily from [Debian iso-codes](https://salsa.debian.org/iso-codes-team/iso-codes):

- **ISO 639** — Language codes (parts 1, 2, 3, 5)
- **ISO 3166** — Country and subdivision codes (parts 1, 2, 3)
- **ISO 4217** — Currency codes
- **ISO 15924** — Script codes
- **QUDT** — Units of measurement

Translations will be added in a subsequent phase using the Debian iso-codes translation files as the baseline.
